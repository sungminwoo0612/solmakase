# 2. 서비스 기획

---

## 2.1 메뉴 구조도

```
Solmakase
├── 홈 (/)
│   └── 서비스 소개 및 시작하기
├── 요구사항 입력 (/requirements)
│   ├── 설문조사 입력
│   ├── 문서 업로드
│   ├── 채팅 입력
│   └── 전문가 모드
├── 분석 결과 (/analysis)
│   ├── 요구사항 요약
│   ├── 추출된 정보 확인
│   └── 추가 질문/수정
├── 인프라 설계 (/design)
│   ├── 온프레미스 설계안
│   ├── 클라우드 설계안
│   ├── 비용 비교
│   └── 프롬프트 기반 수정
├── 배포 및 모니터링 (/deployment)
│   ├── 배포 실행
│   ├── 인프라 대시보드 (React Flow)
│   ├── 리소스 모니터링
│   └── 테스트 실행
└── 결과물 다운로드 (/downloads)
    ├── IaC 코드
    ├── 인프라 기획서
    └── 견적서
```

---

## 2.2 기능 정의서

| 기능ID | 대상 | 기능명 | 기능 설명 |
|--------|------|--------|-----------|
| **F-100** | 사용자 | 요구사항 설문조사 입력 | 서비스 종류, 온프레미스/클라우드, 규모, 예산, 운영 인력 등 설문조사 형식으로 입력 |
| **F-101** | 사용자 | 문서 업로드 | PDF, docx, pptx, 한글 문서 등 다양한 형식의 견적서/요구사항 문서 업로드 |
| **F-102** | 사용자 | 채팅 입력 | 부족한 정보를 보완하기 위한 대화형 챗봇 인터페이스 |
| **F-103** | 사용자 | 전문가 모드 | 기능별 선택 버튼을 통한 요구사항 입력 (로그인, 파일 서버, 객체 스토리지 등) |
| **F-110** | 시스템 | 문서 분석 (RAG) | 업로드된 문서를 RAG를 활용하여 분석하고 요구사항 추출 |
| **F-111** | 시스템 | 요구사항 구조화 | Agent LLM을 통한 요구사항 추출 및 구조화된 데이터 생성 |
| **F-112** | 시스템 | 기술 스택 추천 | 외부 툴 연동을 통한 최적의 기술 스택 추천 |
| **F-120** | 사용자 | 인프라 설계안 조회 | 온프레미스/클라우드 각각에 대한 최적의 인프라 아키텍처 제안 확인 |
| **F-121** | 사용자 | 비용 산정 조회 | 예상 비용 및 리소스별 비용 내역 확인 |
| **F-122** | 사용자 | 인프라 기획안 조회 | 자동 생성된 인프라 구현 기획안 확인 |
| **F-130** | 시스템 | IaC 코드 생성 | Terraform, Ansible, Kubernetes Manifest 등 IaC 코드 자동 생성 |
| **F-131** | 시스템 | 코드 검증 | 생성된 IaC 코드의 문법 및 구조 검증 |
| **F-132** | 사용자 | 프롬프트 기반 코드 수정 | 자연어 프롬프트를 통한 IaC 코드 수정 요청 |
| **F-133** | 사용자 | 변경사항 미리보기 | 코드 수정 전 변경사항 미리보기 및 적용 확인 |
| **F-140** | 사용자 | 인프라 배포 실행 | 생성된 IaC 코드를 기반으로 실제 인프라 환경 배포 |
| **F-141** | 사용자 | 배포 상태 조회 | 배포 진행 상황 및 상태 실시간 확인 |
| **F-142** | 사용자 | 인프라 구조 시각화 | React Flow를 활용한 전체 인프라 구조 시각화 |
| **F-143** | 사용자 | 리소스 상세 정보 조회 | 각 리소스별 상세 정보 및 설정 확인 |
| **F-144** | 사용자 | 네트워크 커넥션 시각화 | 리소스 간 네트워크 연결 관계 시각화 |
| **F-145** | 사용자 | 실시간 모니터링 | 리소스 상태, 메트릭, 비용 등 실시간 모니터링 |
| **F-146** | 사용자 | 테스트 실행 | 네트워크 연결 테스트, 헬스 체크, 테스트 코드 실행 |
| **F-150** | 사용자 | IaC 코드 다운로드 | 생성된 Terraform, Ansible 등 IaC 코드 파일 다운로드 |
| **F-151** | 사용자 | 인프라 기획서 다운로드 | 인프라 구현 기획서 PDF 다운로드 |
| **F-152** | 사용자 | 견적서 다운로드 | 예상 리소스 자원 정리 및 견적서 다운로드 |

## 2.3 정책 정의서

| 구분 | 정책명 | 정책내용 | 세부 기준/설명 |
|------|--------|----------|----------------|
| **보안** | 파일 업로드 제한 | 업로드 가능한 파일 형식 및 크기 제한 | PDF, docx, pptx, 한글 문서만 허용, 최대 50MB |
| **보안** | 문서 보관 기간 | 업로드된 문서의 보관 기간 | 분석 완료 후 7일간 보관, 이후 자동 삭제 |
| **보안** | 데이터 암호화 | 사용자 데이터 암호화 저장 | 민감 정보는 암호화하여 저장 |
| **성능** | 분석 응답 시간 | 요구사항 분석 완료 시간 | 일반적인 경우 5분 이내, 복잡한 경우 10분 이내 |
| **성능** | 배포 타임아웃 | 인프라 배포 최대 소요 시간 | 배포 시작 후 30분 이내 완료, 초과 시 알림 |
| **사용성** | 입력 방식 선택 | 사용자가 원하는 방식으로 입력 가능 | 설문조사, 문서 업로드, 채팅, 전문가 모드 중 선택 |
| **사용성** | 실시간 피드백 | 작업 진행 상황 실시간 표시 | 분석, 코드 생성, 배포 등 각 단계별 진행률 표시 |
| **비용** | 비용 산정 정확도 | 예상 비용과 실제 비용의 오차 범위 | ±20% 이내의 오차 범위 목표 |
| **비용** | 무료 체험 제공 | 초기 사용자 무료 체험 제공 | 첫 인프라 설계 및 코드 생성 무료 제공 |
| **품질** | 코드 검증 필수 | 생성된 모든 IaC 코드 검증 필수 | 문법 검증, 베스트 프랙티스 검증 통과 필수 |
| **품질** | 배포 전 확인 | 배포 실행 전 사용자 확인 필수 | 배포 실행 전 변경사항 확인 및 승인 절차 |
| **지원** | 에러 처리 | 에러 발생 시 명확한 메시지 제공 | 사용자 친화적인 에러 메시지 및 해결 방법 제시 |
| **지원** | 버전 관리 | IaC 코드 버전 관리 | 모든 코드 변경 이력 추적 및 롤백 가능 |
