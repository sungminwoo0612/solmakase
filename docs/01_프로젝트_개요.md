# 1. 프로젝트 개요

## 1.1 추진배경 및 목적

### 서비스 대상 (페르소나)

#### 페르소나 1: 비IT 직군 기획자/의사결정자
- **특성**: IT 지식이 제한적이지만 비즈니스 요구사항을 명확히 정의할 수 있는 기획자 또는 의사결정자
- **상황**: 아이디어나 요구사항은 있지만 기술적 구현 방법을 모르는 상황
- **니즈**: 기술 용어 없이도 자신의 의도를 전달하여 최적의 인프라 아키텍처를 얻고 싶음
- **목표**: 빠른 의사결정을 위한 인프라 옵션 비교 및 견적 확인

#### 페르소나 2: 바이브 코딩 개발자
- **특성**: LLM 도구를 활용하여 빠르게 프로토타입을 개발한 개발자 (풀스택 지식은 제한적)
- **상황**: PoC나 MVP는 완성했지만 프로덕션 환경 구축에 대한 경험이 부족
- **니즈**: 개발한 서비스를 안정적으로 운영할 수 있는 인프라 환경이 필요
- **목표**: 최소한의 학습으로 프로덕션 레벨의 인프라를 구축하고 싶음

#### 페르소나 3: 스타트업/소규모 팀
- **특성**: 제한된 예산과 인력으로 빠르게 서비스를 런칭해야 하는 스타트업 또는 소규모 팀
- **상황**: 인프라 전문 인력이 없거나 외주 비용이 부담되는 상황
- **니즈**: 비용 효율적이면서도 확장 가능한 인프라 아키텍처가 필요
- **목표**: 예산 내에서 최적의 인프라를 구축하고, 향후 확장 시에도 유연하게 대응하고 싶음

### 추진배경

**배경 1: 인프라 아키텍처 구성의 복잡성**
- 프로덕션 레벨의 인프라 아키텍처 구성에는 많은 시간과 검토 작업이 필요함
- 이러한 과정을 자동화하여 고객이 빠르게 프로덕션 레벨의 인프라를 실험하고, 의사결정을 할 수 있도록 지원
- 비IT 직군의 기획자 혹은 의사결정자라도 "오마카세"처럼 의도한 바를 짚어내어 기술적으로 최적의 인프라 아키텍처를 제안

**배경 2: 개발 진입장벽 하락과 운영의 문제**
- LLM의 눈부신 발전으로 바이브 코딩 등의 방법론이 등장함
- 이에 따라 개발의 진입 장벽은 낮아졌고, 다양한 소프트웨어들이 등장함
- 비개발자가 자신의 아이디어를 구현하는 속도는 빨라졌지만 여전히 운영에 대한 문제가 있음
- 본 서비스는 빠르게 만든 PoC 프로젝트를 위한 맞춤형 솔루션을 제공하기 위함

### 목적
**Solmakase**는 사용자의 요구사항을 분석하여 최적의 인프라 아키텍처를 자동으로 설계하고, IaC(Infrastructure as Code) 코드를 생성하여 실제 배포까지 지원하는 AI 기반 인프라 오마카세 서비스입니다.

## 1.2 주요 기능

### 핵심 기능 3가지

#### 1. 요구사항 수집 및 분석 (Requirement Collection & Analysis)
다양한 형식의 정보를 수집하여 사용자의 인프라 요구사항을 분석합니다.

**1.1 다중 입력 채널**
- **설문조사 기반 입력**: 서비스 종류, 온프레미스/클라우드 선호도, 서비스 규모, 예산, 운영 인력 여부, 운영 환경 특이사항 등
- **문서 기반 입력**: PDF, docx, pptx, 한글 문서 등 다양한 형식의 견적서/요구사항 문서 업로드
- **대화형 입력**: 부족한 정보를 보완하기 위한 챗봇 인터페이스
- **전문가 모드**: 기존 SI 견적 경험이 있는 사용자를 위한 기능별 선택 인터페이스
  - 로그인 기능, 파일 서버 기능, 객체 스토리지 기능 등

**1.2 AI 기반 분석**
- RAG(Retrieval-Augmented Generation)를 활용한 문서 분석
- Agent LLM을 통한 요구사항 추출 및 구조화
- 외부 툴 제한적 연동을 통한 기술 스택 추천

#### 2. 인프라 설계 및 코드 생성 (Infrastructure Design & Code Generation)
분석된 요구사항을 기반으로 최적의 인프라 아키텍처를 설계하고 IaC 코드를 자동 생성합니다.

**2.1 인프라 기획안 작성**
- 온프레미스/클라우드 각각에 대한 최적의 인프라 아키텍처 제안
- 인프라 구현 기획안 자동 생성
- 예상 비용 산정

**2.2 IaC 코드 생성**
- Terraform, Ansible, Kubernetes Manifest 등 다양한 IaC 도구 지원
- 생성된 코드의 검증 및 최적화
- 버전 관리 및 이력 추적

**2.3 실시간 수정 지원**
- 프롬프트 기반 IaC 코드 수정
- 변경사항 미리보기 및 적용 확인

#### 3. 배포 및 모니터링 (Deployment & Monitoring)
생성된 인프라를 실제로 배포하고 모니터링할 수 있는 대시보드를 제공합니다.

**3.1 자동 배포**
- 생성된 IaC 코드를 기반으로 실제 인프라 환경 배포
- 배포 상태 추적 및 알림

**3.2 시각화 대시보드**
- React Flow를 활용한 전체 인프라 구조 시각화
- 각 리소스별 상세 정보 인터페이스
- 네트워크 커넥션 시각화
- 실시간 상태 모니터링

**3.3 운영 지원 기능**
- 테스트 코드 실행
- 네트워크 연결 테스트
- 리소스 사용량 모니터링
- 비용 추적

**3.4 결과물 제공**
- IaC 코드 다운로드
- 인프라 구현 기획서 제공
- 예상 리소스 자원 정리 및 견적서 생성

### 향후 고려 사항
- Solmakase 서비스 상에서 직접 운영할 수 있는 기능 지원
